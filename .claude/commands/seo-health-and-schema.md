# /seo-health-and-schema

## Scope

Three SEO improvements that share infrastructure:
1. New `/seo-health` route showing per-dealership location page health (title, meta, schema, GBP linkage, Core Web Vitals).
2. Schema generator that outputs ready-to-deploy LocalBusiness, AutoDealer, and Event schema markup tied to submissions.
3. Wire the existing /tools utilities into the relevant /upload steps so they stop being isolated.

Do not modify /admin queue logic, /calendar, or /analytics beyond linking to /seo-health where relevant.

## Context

- Live reference: https://asbury-social-hub.vercel.app/tools has working SERP Preview, Schema Validator, Meta Inspector, PageSpeed buttons but they are disconnected from actual dealership URLs.
- Each dealership has a website (Dealer.com, DealerOn, WebFly, etc.). The hub does not currently track per-dealership website health.
- Schema generated by the tool must be production-grade per Schema.org and Google's structured data requirements (validators must pass without warnings).

## Required reading before any code change

1. Read `/app/tools/page.tsx` to inventory the existing SERP Preview, Schema Validator, Meta Inspector, PageSpeed components.
2. Read the Dealership model. Add `website_url` field if it does not exist (default to inferred URL pattern; require admin to verify in /settings).
3. Read Google Search Console API documentation and PageSpeed Insights API documentation.

## Implementation requirements

### Part 1 - /seo-health dashboard

#### Data model

- New table `seo_health_snapshots`:
  - `id`, `dealership_id`, `page_url`, `page_type` enum (`home`, `new_inventory`, `used_inventory`, `service`, `parts`, `finance`, `about`, `vdp`, `other`)
  - `title_tag`, `title_tag_length`, `meta_description`, `meta_description_length`, `h1_text`, `h1_count`
  - `has_local_business_schema`, `has_auto_dealer_schema`, `has_faq_schema`, `has_breadcrumb_schema`, `schema_errors` (string[])
  - `gbp_linked` (bool, true if page links to or is referenced by the dealership's GBP profile)
  - `sitemap_indexed` (bool)
  - `core_web_vitals` jsonb: `{ lcp, inp, cls, ttfb }` plus pass/fail per metric
  - `last_crawled_at`, `last_gsc_data_at`
  - `gsc_impressions_28d`, `gsc_clicks_28d`, `gsc_avg_position_28d`
- New table `dealership_pages`: per-dealership list of URLs to monitor. Admins seed this with the dealership's standard pages.

#### Crawler

- Cron job runs nightly per dealership. For each URL in `dealership_pages`:
  - Fetch HTML (use `undici` or `node-fetch` with a 10s timeout and a `User-Agent: Asbury-Social-Hub-Crawler/1.0`).
  - Parse with `cheerio` to extract title, meta, H1, schema scripts.
  - Validate schema with the official Schema.org validator API.
  - Call PageSpeed Insights API for Core Web Vitals (mobile + desktop).
  - Call Search Console API for query/click data (requires per-property OAuth in /settings).
  - Persist as a new snapshot row. Keep last 90 days of snapshots per page for trend analysis.

#### /seo-health UI

- Default view: scoreboard with one row per dealership, columns:
  - Pages monitored
  - Pages with schema issues
  - Pages failing Core Web Vitals
  - Avg position (28d)
  - Clicks (28d)
  - Last updated
- Click a row to drill into per-page detail.
- Per-page detail shows: latest snapshot, 30-day sparklines for impressions/clicks/position, issue list with severity, "Re-crawl now" button.
- Issue list categorizes problems: `missing_schema`, `title_too_long`, `meta_too_short`, `cwv_failing_lcp`, etc. Each issue links to a relevant Search Central documentation URL.

#### GSC integration in /settings

- Add "Google Search Console" subsection per dealership: OAuth connect with `https://www.googleapis.com/auth/webmasters.readonly` scope. Store property URL and refresh token.

### Part 2 - Schema generator

#### Generated schema types (closed list)

- `LocalBusiness` (per dealership)
- `AutoDealer` (per dealership; subtype of LocalBusiness)
- `Event` (when /upload Step 3 content_type is `event_promotion`)
- `Offer` (when /upload Step 3 content_type is `lease_special` or `financing_offer`)
- `FAQPage` (manually built in /seo-health per dealership)

#### Generator API

- Create `/lib/schema/generate.ts` exporting one function per schema type. Inputs are typed; outputs are valid JSON-LD objects.
- LocalBusiness/AutoDealer must include: `@context`, `@type`, `name`, `image`, `url`, `telephone`, `address` (PostalAddress with all fields), `geo` (latitude/longitude), `openingHoursSpecification` (array), `priceRange`, `sameAs` (social profile URLs), `aggregateRating` if reviews exist (pull from GBP).
- AutoDealer additionally includes `brand` (Honda/BMW/etc.) and `makesOffer` (array referencing the inventory feed).
- Event must include `name`, `startDate` (ISO 8601 with timezone), `endDate`, `eventStatus`, `eventAttendanceMode`, `location` (LocalBusiness reference), `image`, `description`, `offers` (if applicable), `organizer`.
- Offer must include `priceCurrency`, `price` or `priceSpecification`, `validFrom`, `validThrough`, `eligibleQuantity` if limited, `availability`.

#### /seo-health schema panel per dealership

- Show the current LocalBusiness/AutoDealer JSON-LD for the dealership.
- "Copy to clipboard" button.
- "Validate" button that submits to the Schema.org validator API and shows results inline.
- Diff view if the live page's schema does not match what the generator produces.

#### Upload integration

- In /upload Step 5, when content_type is `event_promotion`:
  - Inline Event schema generator panel appears.
  - Required fields: start datetime, end datetime, event name, optional offers.
  - On submit, the generated JSON-LD is attached to the submission as `generated_schema` jsonb.
  - On approval, the schema is exposed via a "Schema markup" panel in /admin so the web team can copy it to the corresponding landing page.

### Part 3 - Tools as workflow plumbing

- SERP Preview component: imported into /upload Step 5 when platform = LinkedIn AND content_type = `linkedin_article`. Pre-fill from caption first line and the article URL.
- Schema Validator component: imported into the schema panel above; runs automatically on the generated JSON-LD.
- Meta Inspector component: imported into /seo-health per-page detail view; runs on demand for any monitored URL.
- UTM Builder: already handled by `/upload-flow-overhaul`; no UI needed.
- Refactor /tools page so the SERP Preview etc. components are exported from `/components/tools/` and imported elsewhere. The /tools page itself becomes thin wrappers around those components.

## Out of scope - do NOT

- Do NOT build a full SEO crawler with discovery; only crawl URLs explicitly listed in `dealership_pages`.
- Do NOT integrate with a third-party rank tracker (BrightLocal, Ahrefs, SEMrush) in this command.
- Do NOT auto-fix any SEO issues; report only.
- Do NOT write generated schema to the dealership websites directly; output is copy/paste only.
- Do NOT add Product schema for inventory in this command; vehicle structured data deserves its own dedicated command.
- Do NOT remove the /tools page; it remains as a centralized reference.

## Acceptance criteria

- /seo-health route exists and renders one row per dealership with monitored page count.
- Adding a URL to a dealership's `dealership_pages` and triggering the crawler populates a snapshot within 5 minutes.
- A page with no LocalBusiness schema shows `has_local_business_schema = false` and surfaces a `missing_schema` issue.
- The schema generator outputs valid LocalBusiness JSON-LD that passes Schema.org validator with zero errors.
- Submitting an event_promotion post in /upload generates and stores Event schema with required fields.
- /admin submission detail for an event_promotion submission shows a "Schema markup" copy panel.
- /upload Step 5 with platform = LinkedIn + content_type = linkedin_article shows an inline SERP preview using the article title and meta description.
- PageSpeed Insights data populates within the snapshot for monitored URLs.
